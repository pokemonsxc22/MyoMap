Replace the current email/magic link authentication system in MyoMap with a simple name-based identity system. Here's exactly what to build:

## How it works
- On first visit, show a welcome screen asking for the user's name (just a first name is fine)
- Generate a UUID for them, store it in localStorage as `myomap_user_id`, and store their name as `myomap_user_name`
- Save a record to Supabase in a `users` table: { id (uuid), name (text), created_at }
- On every subsequent visit, read from localStorage — no login screen needed
- All assessments and progress are tied to this UUID (replace any auth user_id references with this UUID)

## Supabase setup
Run this SQL in Supabase SQL editor:
CREATE TABLE IF NOT EXISTS users (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

GRANT ALL ON users TO anon;
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all on users" ON users FOR ALL TO anon USING (true) WITH CHECK (true);

- Clean centered card, teal (#0D9488) accent, dark slate (#1E293B) background
- MyoMap logo/name at top
- Heading: "Welcome to MyoMap"
- Subtext: "Enter your name to get started and track your progress."
- Single input: "Your first name" placeholder
- Button: "Get Started" (teal)
- On submit: generate UUID, save to localStorage AND Supabase users table, then redirect to the intake form
- Validate that the name field is not empty before submitting

## Returning user flow
- On app load, check localStorage for `myomap_user_id` and `myomap_user_name`
- If both exist, skip the name screen entirely and go straight to the app
- Show their name in the dashboard header (e.g. "Welcome back, Sohan")

## Dashboard & assessments

- The dashboard at /dashboard should load assessments filtered by this UUID
- Remove any "sign out" or auth-related UI elements
- Keep all existing assessment, progress, and streak functionality exactly as-is — just swap the user identifier
